🔄 Automation

Control how often Dockpatch checks for image updates and how it applies them automatically.

⏱ Schedule

Set intervals for automatic checking and updating. Set either to 0 to disable that schedule entirely.

How often Dockpatch polls all container images against their registry. 0 = disabled.
How often Dockpatch automatically recreates containers that have updates waiting. 0 = disabled (manual only).
🌙 Maintenance Window

Restrict auto-updates to a specific time window. Leave both fields empty to allow updates at any time. Overnight ranges are supported (e.g. 22:00 → 04:00). Times use the server's clock — set the TZ environment variable to control the timezone.

→
Both fields must be set to activate the window, or both must be empty to deactivate it.

🔑 Registry Authentication

Add credentials so Dockpatch can check and pull images from private registries. For GitHub Container Registry (ghcr.io), use your GitHub username and a Personal Access Token with the read:packages scope.

🏷 Configured Registries

Each registry entry stores a username and a token or password. Tokens are stored on disk and are never transmitted to the browser.

⚠️ Credentials are stored in plain text inside your /data volume. Ensure the volume is not world-readable and the host is adequately secured.
